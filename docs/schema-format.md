# Schema Format Reference

This document covers all schema file formats, types, attributes, and syntax supported by the Metaphor schema system.

## File Types

| Extension | Purpose | Format |
|-----------|---------|--------|
| `*.model.yaml` | Entity definitions, shared types, enums | YAML (recommended) |
| `*.hook.yaml` | Entity lifecycle behaviors, state machines, rules, triggers | YAML |
| `*.workflow.yaml` | Multi-step business processes (Saga pattern) | YAML |
| `*.model.schema` | Entity definitions (legacy) | Custom DSL |
| `*.workflow.schema` | Workflow definitions (legacy) | Custom DSL |

Schema files are discovered automatically in the module's `schema/` directory:

## YAML Model Format

The `*.model.yaml` file is the primary schema definition format. It supports the following top-level sections:

```yaml
# Data definitions
models:           # Model/table definitions
enums:            # Enumeration types
types:            # Custom reusable type definitions
generators:       # Per-file generator configuration

# DDD extensions (see ddd-guide.md)
entities:         # Enhanced models with behavior
value_objects:    # Domain-specific composite/wrapper types
domain_services:  # Services with dependencies
event_sourced:    # Event sourcing configurations
authorization:    # RBAC/ABAC permissions and roles
usecases:         # Application layer operations
events:           # Domain event definitions
projections:      # CQRS read models
services:         # Application services
handlers:         # Event handlers
subscribes_to:    # Cross-module event subscriptions
integration:      # ACL adapter definitions
presentation:     # HTTP/gRPC config
dtos:             # Request/response definitions
versioning:       # API versioning config
traits:           # Repository trait definitions
```

### Models

Define data models that map to database tables and Rust structs:

```yaml
models:
  - name: Order
    collection: orders          # Database table name
    soft_delete: true           # Enable soft-delete (adds deleted_at field)
    fields:
      id:
        type: uuid
        primary_key: true
      customer_id:
        type: uuid
        required: true
      email:
        type: string
        required: true
        unique: true
      status:
        type: OrderStatus       # Reference to an enum
        required: true
      total:
        type: decimal
      metadata:
        type: json
      created_at:
        type: timestamp
        auto: true              # Auto-generated by database
      updated_at:
        type: timestamp
        auto: true
    relations:
      customer:
        type: Customer
        attributes: ["@one", "@foreign_key(customer_id)", "@on_delete(restrict)"]
      items:
        type: OrderItem[]
        attributes: ["@one_to_many", "@on_delete(cascade)"]
    indexes:
      - fields: [customer_id]
      - fields: [status, created_at]
      - fields: [email]
        unique: true
```

### Field Properties

| Property | Type | Description |
|----------|------|-------------|
| `type` | string | Field type (primitive, enum reference, or custom type) |
| `required` | bool | Field cannot be null (default: `false`) |
| `primary_key` | bool | Mark as primary key (default: `false`) |
| `unique` | bool | Add unique constraint (default: `false`) |
| `auto` | bool | Auto-generated value (default: `false`) |
| `default` | any | Default value for the field |

### Enums

Define enumeration types:

```yaml
enums:
  - name: OrderStatus
    variants:
      - Pending
      - Confirmed
      - Shipped
      - Delivered
      - Cancelled

  - name: UserRole
    values: [admin, editor, viewer]    # Alternative short syntax
```

Enum names are automatically converted to **PascalCase** in generated Rust code:

```yaml
# Schema
enums:
  - name: relationship_type
    values: [lead, customer, partner]

# Generated Rust
# pub enum RelationshipType { Lead, Customer, Partner }
```

### Shared Types

Define reusable type compositions:

```yaml
types:
  - name: AuditFields
    fields:
      created_at:
        type: timestamp
        auto: true
      updated_at:
        type: timestamp
        auto: true
      created_by:
        type: uuid
      updated_by:
        type: uuid
```

### Generators Configuration

Control which generators run at the file level:

```yaml
generators:
  enabled: [rust, sql, repository]     # Whitelist mode
  disabled: [cqrs, projection]         # Blacklist mode
  cqrs: true                           # Explicit opt-in for a specific target
```

## Field Attributes

Used in the legacy DSL format and in relation `attributes` arrays:

| Attribute | Description | Example |
|-----------|-------------|---------|
| `@id` | Primary key | `id uuid @id` |
| `@unique` | Unique constraint | `email string @unique` |
| `@required` | Not nullable | `name string @required` |
| `@default(value)` | Default value | `@default(now)`, `@default(uuid)`, `@default(active)` |
| `@min(n)` | Minimum length/value | `@min(3)` |
| `@max(n)` | Maximum length/value | `@max(255)` |
| `@pattern(regex)` | Regex validation pattern | `@pattern([a-z0-9-]+)` |
| `@foreign_key(ref)` | Foreign key reference | `@foreign_key(User.id)` |
| `@soft_delete` | Enable soft-delete on model | - |
| `@hashed` | Field is stored hashed (e.g., passwords) | `password string @hashed` |
| `@audit_metadata` | JSONB audit metadata field | - |

### Foreign Key Actions

| Attribute | Values | Description |
|-----------|--------|-------------|
| `@on_delete(action)` | `cascade`, `restrict`, `set_null`, `set_default` | Action when parent is deleted |
| `@on_update(action)` | `cascade`, `restrict`, `set_null`, `set_default` | Action when parent is updated |

## Relation Attributes

| Attribute | Description |
|-----------|-------------|
| `@one` | One-to-one relationship |
| `@many` | One-to-many relationship |
| `@one_to_many` | One-to-many (explicit) |
| `@many_to_many` | Many-to-many relationship |
| `@belongs_to` | Inverse relationship |
| `@references(column)` | Custom referenced column |
| `@join_table(name)` | Many-to-many join table name |
| `@join_fk(column)` | Many-to-many FK column |
| `@foreign_key(column)` | Explicit FK column name |

### Relation Examples

```yaml
relations:
  # One-to-one
  profile:
    type: Profile
    attributes: ["@one"]

  # One-to-many with cascade delete
  items:
    type: OrderItem[]
    attributes: ["@one_to_many", "@on_delete(cascade)"]

  # Many-to-many with custom join table
  roles:
    type: Role[]
    attributes: ["@many_to_many", "@join_table(user_roles)", "@join_fk(user_id)"]

  # Belongs-to with foreign key
  customer:
    type: Customer
    attributes: ["@belongs_to", "@foreign_key(customer_id)", "@on_delete(restrict)"]
```

## Indexes

In YAML format:

```yaml
indexes:
  - fields: [email]                    # Single-field index
    unique: true
  - fields: [status, created_at]       # Composite index
  - fields: [customer_id]
```

In legacy DSL format:

```
indexes {
    @@index(email)
    @@index(status, created_at)
    @@unique(resource, action)
}
```

### Partial Indexes

Use the `@where` attribute to emit a partial index â€” the predicate is appended
verbatim as a SQL `WHERE` clause:

```yaml
indexes:
  - fields: [email]
    unique: true
    attributes: ['@where(deleted_at IS NULL)']
```

**Audit-metadata rewriting.** When the model carries an `@audit_metadata`
JSONB field, audit keys (`created_at`, `updated_at`, `deleted_at`,
`created_by`, `updated_by`, `deleted_by`) are stored *inside* that JSONB
column, not as real columns. To keep the predicate readable, bare
identifiers matching those keys are rewritten to their JSONB form
(`(metadata->>'deleted_at')`). String literals and identifiers already
qualified (e.g. `t.deleted_at`) are left alone, and a model that declares
one of those names as a real column shadows the rewrite.

```yaml
# Authored:
attributes: ['@where(deleted_at IS NULL)']

# Emitted (model has @audit_metadata field `metadata`):
WHERE (metadata->>'deleted_at') IS NULL
```

## Soft Delete

Enable soft-delete on a model to automatically generate:
- A `deleted_at` timestamp field
- Trash listing endpoints
- Restore endpoint
- Repository methods with soft-delete filtering

```yaml
models:
  - name: User
    collection: users
    soft_delete: true
    fields:
      # ... deleted_at is added automatically
```

## Hook YAML Format

The `*.hook.yaml` files define entity lifecycle behaviors:

```yaml
hooks:
  OrderHooks:
    entity: Order

    states:
      - name: pending
        initial: true
        on_enter:
          - action: send_notification
            params: { template: "order_created" }
      - name: confirmed
      - name: shipped
      - name: delivered
        final: true
      - name: cancelled
        final: true

    transitions:
      - from: pending
        to: confirmed
        guard: "has_payment"
        roles: [admin, manager]
      - from: confirmed
        to: shipped
        roles: [admin, warehouse]
      - from: shipped
        to: delivered
      - from: [pending, confirmed]
        to: cancelled
        roles: [admin, customer]

    rules:
      - name: minimum_order_amount
        condition: "order.total >= 10.00"
        error_code: "ORDER_BELOW_MINIMUM"
        message: "Order total must be at least $10.00"

    permissions:
      pending:
        editable_fields: [items, shipping_address]
      confirmed:
        editable_fields: [shipping_address]
      shipped:
        editable_fields: []

    computed:
      - name: total
        expression: "sum(items.price * items.quantity)"
        type: decimal

    triggers:
      - event: on_transition
        from: pending
        to: confirmed
        action: charge_payment
      - event: on_enter
        state: shipped
        action: send_shipping_notification
```

## Workflow YAML Format

The `*.workflow.yaml` files define multi-step business processes following the Saga pattern:

```yaml
workflows:
  OrderProcessing:
    description: "End-to-end order fulfillment"
    steps:
      - name: validate_order
        action: validate
        entity: Order
        on_failure: abort

      - name: reserve_inventory
        action: reserve
        entity: Inventory
        compensation: release_inventory

      - name: charge_payment
        action: charge
        entity: Payment
        compensation: refund_payment

      - name: ship_order
        action: ship
        entity: Order
        compensation: cancel_shipment

    on_complete:
      - send_confirmation_email
    on_failure:
      - notify_support
```

## Legacy DSL Format

The legacy `.model.schema` format uses a custom DSL syntax. It is still supported for backward compatibility but YAML is recommended for new schemas.

### Model Definition

```
model User {
    collection: users

    fields {
        id          uuid        @id @default(uuid)
        email       email       @unique @required
        username    string      @unique @required @min(3) @max(50)
        password    string      @required @min(8)
        status      Status      @default(active)
        created_at  datetime    @default(now)
        updated_at  datetime?
        deleted_at  datetime?
    }

    relations {
        profile     Profile     @one
        roles       Role[]      @many_to_many
    }

    indexes {
        @@index(email)
        @@index(username)
        @@index(status, created_at)
    }
}
```

### Enum Definition

```
enum Status {
    active      @default @label("Active User")
    inactive    @label("Inactive User")
    pending     @label("Pending Verification")
    suspended   @label("Suspended Account")
}
```

### Workflow Definition (Legacy)

```
workflow OrderApproval {
    states {
        pending     @initial
        approved
        rejected    @final
    }

    transitions {
        pending -> approved [role: manager]
        pending -> rejected [role: manager]
    }

    rules {
        rule minimum_amount {
            condition: "order.total >= 100"
            error: "Orders below $100 do not require approval"
        }
    }
}
```

### Key Differences from YAML

| Feature | Legacy DSL | YAML |
|---------|-----------|------|
| Nullable types | `datetime?` suffix | Omit `required: true` |
| Attributes | Inline `@attr` syntax | `attributes` array or field properties |
| Enums | `enum Name { ... }` block | `enums:` list with `variants` |
| Relations | Inline in `relations { }` block | `relations:` mapping |
| DDD extensions | Not supported | Full support (entities, VOs, services, etc.) |
| Shared types | Not supported | `types:` section |
