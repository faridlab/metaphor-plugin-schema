# Codegen triage: generated-but-undeclared module wiring

Filed 2026-07-02. Found while wiring `backbone-catalog` after a `schema generate`
run. The generator emits several module files on disk but never declares them in
the crate root (or the layer `mod.rs`), so they silently fall out of the compile.
The affected crate still builds ‚Äî because the orphaned modules are simply never
reached ‚Äî which is exactly why the gap is easy to miss.

Evidence below is from `backbone-catalog` (8 entities) generated by the current
`metaphor-schema`. Repro at the bottom.

## Symptom

`src/lib.rs` (generated, "do not edit manually") declares only:

```
pub mod domain;
pub mod infrastructure;
pub mod application;
pub mod presentation;
pub mod seeders;
```

But the generator also wrote, and left **undeclared**:

| Generated tree | Emitted by | Declared anywhere? |
|---|---|---|
| `src/handlers/` (`AppState` DI container) | `src/generators/app_state.rs`, `handlers_module.rs` | ‚úó not in `lib.rs` |
| `src/routes/` (stateless/stateful composers) | `src/generators/routes_composer.rs` | ‚úó not in `lib.rs` |
| `src/exports/` (public cross-module API) | `src/generators/export.rs` | ‚úó not in `lib.rs` |
| `src/application/usecases/` | `src/generators/cqrs.rs` | ‚úó not in `application/mod.rs` |
| `src/application/auth/` | (auth generator) | ‚úó not in `application/mod.rs` |
| `src/application/bulk_operations/` | (bulk generator) | ‚úó not in `application/mod.rs` |
| `src/application/subscriptions/` | `src/generators/event_subscription.rs` | ‚úó not in `application/mod.rs` ‚Äî **and does not compile, see Gap B** |

Net effect: `CatalogModule::all_crud_routes()` works, but the generated
`create_stateless_routes` / `create_stateful_routes` composers, the `AppState`
DI path, the public `exports` API, and the use-case/auth/bulk layers are all
dead until someone hand-declares them.

## Gap A ‚Äî `lib.rs` / `application/mod.rs` templates omit sibling modules

Root cause: the crate-root template is emitted by
`src/generators/module.rs:78-83`, which hard-codes exactly five `pub mod` lines
and has no knowledge of what the `app_state`, `routes_composer`, `export`, `cqrs`,
auth, bulk, and `event_subscription` generators separately write.

```
src/generators/module.rs:78  writeln!(output, "pub mod domain;")
src/generators/module.rs:79  writeln!(output, "pub mod infrastructure;")
src/generators/module.rs:80  writeln!(output, "pub mod application;")
src/generators/module.rs:81  writeln!(output, "pub mod presentation;")
src/generators/module.rs:83  writeln!(output, "pub mod seeders;")   // <-- stops here
```

**Proposed fix:** whichever generators emit `handlers/`, `routes/`, `exports/`
must also contribute their `pub mod` line to `lib.rs`, and the
`usecases/auth/bulk_operations/subscriptions` generators must contribute theirs to
`application/mod.rs`. Options, cheapest first:
1. Have `module.rs` emit the extra `pub mod` lines conditionally on which
   generators are enabled for the run (single source, but couples `module.rs` to
   the others).
2. Give each generator an "append a `pub mod X;` to the parent `mod.rs`/`lib.rs`"
   step, merged via the existing custom-block merge machinery
   (`src/commands/schema/merge/custom_blocks/mod.rs`).

Either way the declaration must land **inside a `// <<< CUSTOM ‚Ä¶ // END CUSTOM`
block** so a later regen preserves it (that is the manual workaround applied to
`backbone-catalog` today ‚Äî see `backbone-catalog/src/lib.rs` and
`src/application/mod.rs`).

## Gap B ‚Äî `subscriptions/registry.rs` emits an incomplete type alias

`src/application/subscriptions/registry.rs:15` (generated) is:

```rust
pub type SubscriptionRegistry = SubscriberRegistry;   // ‚úó E0107
```

but the upstream type is generic:

```
backbone-messaging/src/subscriber.rs:131
pub struct SubscriberRegistry<Event: Clone + Send + Sync + 'static> { ‚Ä¶ }
```

So the alias fails with **E0107 (missing generic argument)** the moment the module
is declared. This is emitted by `src/generators/event_subscription.rs:126`:

```
writeln!(output, "pub type SubscriptionRegistry = SubscriberRegistry;")
```

**Proposed fix:** emit a concrete event generic. The module already generates a
per-entity domain-event set (`domain::event::*`), so either:
- emit a module-wide event enum and alias `SubscriberRegistry<CatalogEvent>`, or
- make `SubscriptionRegistry` itself generic: `pub type SubscriptionRegistry<E> = SubscriberRegistry<E>;`

Until fixed, `application/subscriptions` cannot be wired; `backbone-catalog`
leaves it undeclared with an explanatory note.

## Gap C ‚Äî `src/bin/seeder.rs` is a binary target in a lib-only module

`module`-type projects MUST be library-only (no binary target ‚Äî see the module
`CLAUDE.md`). But the generator writes `src/bin/seeder.rs`, which Cargo
auto-discovers as a `seeder` binary (the crate has `[lib]` and no `[[bin]]`, so
`autobins` is on). This silently gives every generated module a binary target.

**Proposed fix:** emit the seeder as a feature-gated example (`examples/`), a
`[[bin]]` guarded behind a `seeder` feature, or a library function invoked by an
existing CLI ‚Äî not an always-on `src/bin/*.rs`.

## Repro

```bash
cd frameworks/metaphora/backbone-catalog
metaphor schema schema generate      # regenerate from schema/models/*.model.yaml
grep -nE 'pub mod' src/lib.rs         # note: no handlers/routes/exports
grep -nE 'pub mod' src/application/mod.rs   # note: only service + validator
# declare application/subscriptions and observe E0107:
#   pub type SubscriptionRegistry = SubscriberRegistry;
cargo check
```

## Status

- **Workaround applied** in `backbone-catalog` (CUSTOM-block declarations in
  `lib.rs` + `application/mod.rs`; `subscriptions` left out pending Gap B).
- **Generator fix** (Gaps A/B/C) ‚Äî open, belongs in this crate's generators.
